AFS-DEA platform manual

1.Introduction

AFS-DEA is a differential analysis and feature selection platform for gene expression data. This platform using Django as the development framework. The page uses HTML, CSS and JQuery to design an intuitive and user-friendly visual web interface. The underlying algorithm is written mainly in Python 3, integrates multiple base classifiers such as randomforest (RF), linear discriminant analysis(LDA), logisticregression (LR), neural network (NN) and k-nearest-neighbor(KNN). In addition, the information is stored in a relational database implemented by MySQL. AFS-DEA is an open source software, you can download the source code from http://bio-nefu.com/afs-dea/code. We also provide a manual and testing data.

2.Installation tutorial

2.1 Create a virtual environment

If you do not have pip3, you can install it with the following command

sudo apt install python3-pip

First install the virtual environment, the command is as follows

xxxxxxxxxx
sudo pip3 install virtualenv

The virtual environment is actually a copy of the real python environment, so that we install the package in the copied python environment without affecting the real python environment. By establishing multiple virtual environments, the development of projects in different virtual environments realizes the isolation between projects.

Next, install the virtual environment extension package. 

xxxxxxxxxx
sudo pip3 install virtualenvwrapper

Modify the configuration file .bashrc in the user's home directory and add the following content:

xxxxxxxxxx
export WORKON_HOME=$HOME/.virtualenvs
export VIRTUALENVWRAPPER_PYTHON=/usr/bin/python3
source /usr/local/bin/virtualenvwrapper.sh

Use the source .bashrc command to make the configuration file effective.

Now you can create a virtual environment, for example, create a virtual environment named django_1.

xxxxxxxxxx
mkvirtualenv django_1

If the virtual environment is not created successfully, you can upgrade the pip package management tool and re-create the virtual environment:

xxxxxxxxxx
sudo pip3 --default-timeout=10000 install --upgrade pip

2.2 Install dependencies

The packages required for AFS-DEA operation can be installed with the following code

xxxxxxxxxx
pip install -r requirements.txt

2.3 Database settings

AFS-DEA uses MySQL database which is the preferred database for web projects.

Firstly, modify file settings.py under untitled1/untitled1. Change the engine to MySQL, provide the connected host HOST, port PORT, database name NAME, user name USER and password PASSWORD as follows. Then save settings and exit.

xxxxxxxxxx
DATABASES = {
'default': {
'ENGINE': 'django.db.backends.mysql',
'NAME': 'db_web_clent', 
'USER': 'root', 
'PASSWORD': '123456', 
'HOST': 'localhost', 
'PORT': '3306', 
}
}

Secondly, due to the Django framework does not automatically generate the database db_web_clent, we need to enter the mysql database to create it ourselves.

If you don't have mysql installed, you can use the following code to install

xxxxxxxxxx
sudo apt install mysql-server

Set password as follows.

xxxxxxxxxx
mysql -uroot -p

The following is to manually create a database db_web_clent.

xxxxxxxxxx
create database db_web_clent charset=utf8

Note: Set the character set to utf8

We provide you with the db.sql in the root directory of the project, you can run the following code to create the table.

xxxxxxxxxx
use db_web_clent;
source db.sql; #must be absolute path

3.Use tutorial

3.1 Basic Settings

Fig. 1 shows the panel for feature selection part of AFS-DEA in website.

Fig. 1. Feature selection model

A.Basic setting B.Dynamic process C.Feature selection result

  • Whether to include test set Yes if the user provides a separate testing set. If set to No, AFS-DEA will automatically divide the input data into training set and testing set.

  • Select file path The input data file, we support the .xlsxfile, with its row and column corresponding to either sample or feature.

  • Select a separate file The testing data provided by users. Only available when Whether to include test set is set to Yes. And the provided data must have the same format with the input data file selected in Select file path.

  • Select your data format The layout of data. If each row of the data in .xlsx file is a sample, then select row: sample,col:feature, and AFS-DEA assumes that the last column corresponds to labels. If each row of the data corresponds to the values of a variable, then select row: feature,col: sample, and AFS-DEA assumes that the second row corresponds to labels.

    • Input data sheet index The data sheet index.
    • Row offset of Data The number of the first row containing data.
    • Col offset of Data The number of the first column containing data.

  • Distance threshold It refers to a cutoff distance dc, which is only used to calculate the local densities of variable importance scores. More details can be seen in our previously proposed clustering algorithm. General value range 0-0.1.

  • Base Classifier The base classifier of the ensemble classifier. AFS-DEA supports randomforest (RF), linear discriminant analysis(LDA), logisticregression (LR), neural network (NN) and k-nearest-neighbor(KNN). By setting Base Classifier to Auto, AFS-DEA will automatically choose the best classifier which has highest out-of-bag classification accuracy.

  • Classifier Number Number of classifiers in the ensemble model.

  • Number of cores Number of cores per node.

3.2 Feature selection operation

3.1.1 Perform computation on the website (URL: http://bio-nefu.com/afs-dea)
  1. Enter the URL in the browser.
  2. Set parameters in main interface. See 3.1 for parameter settings.
  3. Click Run to start computation. When get result! is displayed, the calculation is over. The obtained feature set can be listed in a table for manual selection, as illustrated in Fig. 2F.
  4. Choose one of the feature candidates for validation. Click "Save" to save and click "Plot" to jump.
  5. In the validation interface, you can select another feature for verification, operation like above 4.

If it cannot be displayed normally, please refresh the page

3.1.2 Perform computation in a personal computer or a workstation
  1. Install according to 2 above.
  2. Set parameters in main interface. See 3.1 for parameter settings.
  3. Click Run to start computation. When get result! is displayed, the calculation is over. The obtained feature set can be listed in a table for manual selection, as illustrated in Fig. 2F.
  4. Choose one of the feature candidates for validation. Click "Save" to save and click "Plot" to jump.
  5. In the validation interface, you can select another feature for verification, operation like above 4.

After the basic information is input, with the click of the Run button, the background will check whether the information in the foreground is complete. If not, a corresponding prompt will be shown in the dynamic display module, as shown in Fig. 1B.

Meanwhile, the operation process will also be displayed accordingly in the dynamic display module such as show the specific time and the submit result when the user performs the upload data operation.

After submitting the request, backstage will check whether there is currently only one request to run the algorithm. If it exists for other requests at the same time, a prompt message will be given for asking the request to wait for execution.
